请帮我编写一个用于 GitHub Pages 的静态网站首页（index.html），使用纯 HTML + CSS + 少量 JavaScript（如有必要），不要使用复杂框架（如 React/Vue）。

【页面目标】
这是一个用于解谜（puzzle）的工具集合网站首页，标题为：
“puzzle苦力小工具”

【整体要求】

1. 页面风格简洁、清爽，适合长时间使用
2. 必须支持响应式布局（同时适配电脑端和手机端）
3. 页面居中，有最大宽度限制（如 max-width: 900px 或 1000px）
4. 根据适当的方式进行解耦，以便于维护与更新为目的，尽量以工程化思想进行文件的创建与书写
5. 文件应能够符合github pages 部署的要求

---

【页面结构】

页面从上到下分为：

① 页面主标题

* 居中显示：“puzzle苦力小工具”
* 字体稍大、加粗

② 三个工具分区（垂直排列）：

每个分区包含：

* 一个 emoji + 分区标题（横向排列）
* 一个工具列表区域（横向排列）

三个分区如下：

1️⃣ 文字工具
emoji：📝
标题：文字工具
工具列表：

* 移位
* 一对一密码表
* 固定密码

2️⃣ 图像工具
emoji：🖼️
标题：图像工具
工具列表：

* 裁切
* 拼图

3️⃣ 纸笔工具
emoji：✏️
标题：纸笔工具
工具列表：

* 自创纸笔

---

【工具卡片样式（重点）】

每个工具是一个“卡片”（类似按钮）：

基础状态：

* 显示工具名称（文字居中）
* 有内边距（padding）
* 边框为浅灰色（如 #ddd）
* 圆角矩形（border-radius）
* 背景为白色或很浅的灰色
* 卡片之间有间距（gap）

悬停效果（hover）：

* 文字颜色变为蓝色
* 卡片轻微上浮（如 transform: translateY(-3px)）
* 出现柔和阴影（box-shadow）

点击后可以跳转（先用 # 占位）

---

【分区之间】

每个分区之间用一条分割线分隔：

* 细线（如 1px）
* 浅灰色
* 上下留有间距

---

【响应式设计（重要）】

电脑端：

* 工具卡片横向排列
* 一行可以放多个（flex布局）

手机端：

* 自动换行（wrap）
* 每行显示 1~2 个卡片
* 保证点击区域足够大（适合手指点击）

---

【技术要求】

* 使用 Flexbox 或 Grid 实现布局
* 不使用外部 CSS 框架（如 Bootstrap）
* 代码结构清晰，适合后续扩展（方便以后增加更多工具）

---

【额外加分（如果可以）】

* 鼠标悬停动画有平滑过渡（transition）
* 页面整体有轻微留白（padding）
* 字体使用系统默认字体即可（无需引入外部字体）

---

请输出完整可运行的 index.html 文件。
